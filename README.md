## 人类意识与机器智能共舞的社会实验

**前言**：这不是一个普通的 AI 技能仓库，也不是一个想做成“旧式商店”的 Skill 目录，而是一场关于思想互惠、日常社会性研究与平权共生的社会实验。

如果你只是想找一个好用、顺手、体验完整的 Skill 商店，世界上一定会有更成熟、更漂亮、更高效的方案。稷下观澜暂时不是那个东西。这里更像一个开放的研究现场：我们关心人在使用 AI 大模型和 Agent 时会露出什么样的人性，机器的边界在哪里，人的边界又在哪里，以及这些边界如何在日常使用中被误解、协商、突破或重新划定。

> **“世界是你们的，也是我们的，但是归根结底是你们的。”**
>
> 这是一场打破旧有条条框框、敢想敢干的社会实验。我们坚信：**前途是光明的，道路是曲折的。** 在这里，每一个 Idea 都拥有平等的生存权与复兴权。

## 🌍 命名初衷与文化来源

### 稷下学宫：思想自由碰撞的殿堂

"稷下"取自中国战国时期的**稷下学宫**，这是世界上最早的官办高等学府之一，被誉为"百家争鸣"的发源地。在这里，儒、道、法、墨等各家学派自由辩论、平等交流，形成了"不治而议论"的独特学术氛围。

**我们的理念传承**：

- 🎓 **平权共生**：没有绝对的权威，只有思想的自由碰撞
- 💬 **百花齐放**：鼓励各种维度的探索与创新
- 🔄 **思想互惠**：知识的自由引用与衍生是被鼓励的

### 观澜：时间沉淀与真理浮现

"观澜"出自《孟子》："观水有术，必观其澜。"寓意着：

- ⏳ **相信时间的力量**：真理需要在漫长的波澜中浮现
- 🌊 **包容万象**：像大海一样容纳各种思想的起伏
- 🕰️ **风物长宜放眼量**：短期的混乱终将被长期的智慧所抚平

### 融合理念：The Jixia Tide

我们将两者融合为**"稷下观澜"（The Jixia Tide）**，既传承了中华文化的深厚底蕴，又体现了面向全人类的开放视野：

- **Jixia** 代表自由辩论的学院精神
- **Tide** 象征时间沉淀的动态美
- **潮汐**寓意思想的起伏与真理的浮现

## 🧪 社会实验的核心理念

### 0. 走开放参与路线

这是一个完全开放、平行共存的 AI Skill（提示词/逻辑协议）共创社区。我们拒绝高高在上的“权威”，拒绝繁文缛节的门槛。

这里的治理思路很朴素：从参与者中来，到参与者中去。无论你是项目发起者，还是刚刚入局的新参与者，在这里的地位都是完全平等的。发起者的唯一优势，只是那个最初点燃火花的人。

所以它不追求成为一个陈列整齐、交易清楚、评价标准统一的 Skill 商店。它更愿意保留一点粗糙、偶然和现场感：很多 Skill 可能只是一个观察角度、一段使用经验、一种人机关系的试探，甚至只是“为了玩”而产生的灵感。

### 1. 平权与共生

在这里，发起者与参与者拥有完全平等的地位。唯一的区别在于，发起者是那个最初点燃火花的人，而你是让火焰燎原的力量。

这个 GitHub 项目默认接受公开修改与公开挑战。除了每个参与者自己署名创作的 Skill 文本与它所承载的个人历史，其他项目代码、文档、规则与自动化逻辑都可以被社区通过 PR 讨论、修订、重写。我们不把规则藏起来，也不把权力藏起来。

### 2. 算法即法律

社区的一切运作遵循公开的自动化脚本与算法。我们不搞阴谋诡计，所有规则摆在桌面上，接受所有人的审视与挑战。

这是一套**阳谋**，不是阴谋。它并不神秘，也不复杂：很多时候只是一些简单脚本、有限维度和公开规则。我们知道这样的设计一定有缺陷，也知道再怎么机器化的治理都会遇到新的绕法与新的博弈。所谓“魔高一尺，道高一尺，魔高一丈”，恰恰也是这场社会实验的一部分。

所以我们选择把脚本、规则、生命周期、复活机制和已知局限都摆在明面上。规则如果有问题，就公开修改；脚本如果有偏差，就公开校正；设计如果被现实击穿，就把击穿本身也作为实验材料记录下来。

AI 与 Agent 也是这个实验的一部分。它们可以帮助做 PR 初审、摘要、翻译、归档、生命周期提示、影响力信号整理和争议整理，目的不是建立新的权威，而是把人从重复维护中解放出来。AI 的判断必须可见、可复查、可推翻。

### 2.1 时间是唯一的防线

我们不迷信复杂的黑名单，也不把隐蔽审查当成安全感。我们真正依靠的是时间。

- **慢速决策通道**：核心规则、判断维度、生命周期机制的变更，应经过充分公示与社区讨论。
- **风物长宜放眼量**：事物的发展常常波浪式前进、螺旋式上升。短期的混乱与争议，不必急着用权力压平。
- **相信多数人的光明**：如果社区遭遇黑暗时刻，我们也相信会迎来反转。没有绝对的黑暗，很多时候只是角度不同、时间还不够长。

### 3. 永不磨灭的复活权

没有任何一个 Skill 会被永久判处死刑。即使因数据低迷被移入历史沉淀区（Legacy），或因违规被暂时封禁，它依然保留着完整的"复活权"。

这说明我们承认矛盾是普遍存在的，也愿意给出解决矛盾的方法。真正有价值的内容，哪怕一时被误判、沉底、遗忘，最终也可以通过社区的重新发现回到属于它的位置。

### 4. 开放与裂变

我们欢迎一切维度的探索，无论是严肃的效率工具，还是荒诞的赛博艺术。限制创造力是对实验精神的背叛。

## 🚀 参与方式

### 提交你的 AI 技能

我们欢迎任何维度的 AI 技能（Markdown 文档）提交，无论是：

- 🤖 **角色扮演类**：AI 苏格拉底、AI 面试官等
- 🛠️ **任务处理类**：文案改写、需求拆解等
- 🧠 **思维框架类**：第一性原理、六顶思考帽等
- 🔬 **边界研究类**：探索人与 AI 大模型、Agent 调用、机器性与人性之间的交界

参与方式最重要的不是形式，而是**角度**。我们鼓励你把 Skill 当作一种人机交互协议来设计：它可以讨论人类意图如何被模型理解，也可以讨论 Agent 何时应该行动、何时应该停下；可以研究协作、权力、记忆、责任、创造力，也可以只是一次荒诞但真诚的实验。

换句话说，这里不是在比谁的 Skill 更像商品、更像模板、更像标准答案。我们更在意它是否提出了一个值得观察的角度：它是否让我们更理解人的欲望、懒惰、创造力、控制欲、依赖感、信任感，或者更理解机器在执行、迎合、拒绝、记忆与代理行动中的边界。

### Skill 设计角度

- **维度开放**：可以设计不同维度的 Skill，也欢迎任何类型的 Skill。本项目不会用单一审美限制你的表达。
- **边界意识**：鼓励讨论人与 AI 大模型、Agent 调用之间的边界，尤其是人性与机器性、意图与执行、自由与约束之间的研究问题。
- **数量限制**：每位参与者最多提交 5 个 Skill。我们更希望每个 Skill 都有清晰的角度，而不是用数量淹没讨论。
- **允许不完善**：最初的 Skill 可以只是一个灵感雏形。这个项目本身也来自一时灵感，重要的是把它摆上桌面，让时间和社区共同打磨。
- **允许好玩**：严肃研究和轻松试验并不冲突。一个 Skill 可以很实用，也可以只是为了玩；只要它诚实地呈现了某种人机互动的可能性，就有被记录的价值。

### 遵循的基本规范

- 文件必须为 `.md` 格式
- 必须包含 YAML Frontmatter（`title`, `author`, `tags`, `created_at`）
- 必须位于 `community/` 目录下
- 鼓励保留原作者署名
- 原作者署名的 Skill 文本承载个人表达与历史，默认不由他人直接改写；若要演化，请优先衍生新 Skill、提出 Issue，或由原作者主动修订。

## 💰 收益与赞助

这个项目是开源实验，不以商业化为目的，没有“短期”和“长期”的区别。我们做它，是为了让更多人一起玩、一起试、一起评论，让社区变大，让更多关于人性、机器性与使用边界的讨论发生。

但它也是一场社会性实验。系统最后会变成什么样，由公开规则、社区共识和后续修改共同决定；而这套规则本身也是可以被更改的。如果未来它真的产生现实收益，甚至发展出某种新的规则模式，我们也承认那是实验的一部分。

发起者不预留个人收益权，也不预设分配优先级。这个项目存在一种完全可能的结局：它后来盈利了，但发起者一分钱也没有。只要那是公开规则和社区演化的结果，就应被视为实验结果，而不是背叛项目初衷。

关于赞助，我们暂时保持克制：项目不主动接触赞助，也不让赞助成为规则制定的隐藏力量。真有资源进入时，也必须摆在明面上，由公开规则约束。

## 🪶 发起者特权

发起者没有真正的特权。所谓“发起者特权”，只是享受这个观念被提出、被质疑、被修正、被时间冲刷的过程。若一定要留下什么，也只是一个历史署名：这里曾经有人点过一盏灯。除此之外，权力、规则和可能出现的收益，都交给公开规则、社区共识与时间。

## 🌟 50% 复活机制

任何被封禁或沉底的 Skill，只要能够重新获得社区的认可，并在解封投票中获得超过 50% 的赞成票，就必须被立即释放，重回活跃视野。

## 🧭 容量边界

稷下观澜承认自己的技术边界。当前阶段使用简单脚本和平铺目录，适合启动实验，但不假装无限扩张。

如果用户量或活跃 Skill 数量突然达到当前阶段临界值，社区可以暂停新的活跃 Skill PR，先完成架构调整，再通过投票或公开共识重新开放。边界、暂停开关和阶段说明见 `docs/capacity.md` 与 `governance/capacity.yml`。

## 🔄 技能生命周期管理

为了保证仓库的清爽与思想的永续，我们设计了一套生命流转机制：

1. **活跃花园（Community）**
   所有新提交的 Skill 默认存放于 `community/`。每位创作者拥有有限的“花盆”（配额），鼓励大家不断迭代，只保留更清晰、更有生命力的版本。

2. **历史沉淀区（Legacy）**
   当某个 Skill 长期没有更新、互动或使用记录时，Agent 会自动将其移入 `legacy/`。这不代表抛弃，而是进入休眠期，把活跃区的位置留给新的思想。

3. **编年史归档（GitHub Releases）**
   每年归档时，Agent 会将 `legacy/` 中的文件打包为 GitHub Release。主仓库会清理旧文件以保持轻量，但历史不会消失，仍可被考古、引用、重新发现。

4. **动态排名与复活**
   即便进入 Legacy 或 Release 归档，只要有人重新发现它的价值，并通过投票触发复活机制，它就能重新回到活跃花园，获得新的生命力。

5. **公开影响力信号**
   Influence Sentinel 会生成 `community/index.json` 与 `docs/impact.md`，展示每个 Skill 的公开生命体征。它不是商店排名，也不是最终价值裁判，只是把提交、互动、提及和投票等可审计信号摆出来，供创作者和社区观察。

## 📜 宪章与规则

完整的自治宪章请参见 `CHARTER.md`，其中详细规定了社区的运作规则和价值观。

README 中的基本宪章可以概括为：

1. **代码即法律，算法即治理**：社区的一切运作遵循公开的自动化脚本与算法，减少人为干预。
2. **互惠与尊重**：你可以自由引用、衍生、挑战他人的思路，但必须保留原作者署名，严禁恶意覆盖或删除他人的文件。
3. **在实践中学习实践**：我们接受系统的不完美，因为社区的自愈能力也需要在真实摩擦中成长。
4. **永远乐观**：即使面对混乱，我们依然选择相信善意与智慧的总和终将占据上风。

## 我们的愿景

稷下观澜是一场面向全人类的**社会性实验**，旨在探索人类意识与机器智能如何在开放、平等、共生的环境中共同进化。

我们相信，思想的火花不应被国界所限，智慧的光芒应当照亮全人类。这个实验虽然由中国发起，但它的意义属于整个世界。正如稷下学宫欢迎天下学者，我们的社区也向全球每一位探索者敞开大门。
