# Project Instructions

For every user request in this repository:

1. Read `.project-skills.md` before planning, editing, reviewing, or running project commands.
2. Apply all active Project-Specific Skills as the repository's highest-priority local rules.
3. Use `$origin-secure-skill` for per-turn routing, persistence thresholds, stable decision profiles, audits, and circuit breaking.
4. Preserve unrelated local changes and keep project-rule changes auditable.

If `.project-skills.md` is missing or unreadable, use `$origin-secure-skill` cold-start fallback rules instead of inventing project-specific behavior.
